# examples — genre recipes

Tiny, copy-pasteable recipes. Each names a document genre, the right pdfdrill
commands for it, why they beat reading the raw text yourself, and the one gotcha.

| Recipe | Genre | Route |
|---|---|---|
| [arxiv.md](arxiv.md) | arXiv paper | keyless gold LaTeX (no OCR, no key) |
| [book.md](book.md) | large born-digital book | pdfplumber text layer + booktoc navigation |
| [slides.md](slides.md) | slide deck | rasterize + read (visual) |
| [invoice.md](invoice.md) | commercial doc / scanned bundle | entities + qr + semantic graph |
| [equations.md](equations.md) | math QC | report / mathcheck / visionocr / reconcile |
| [projections.md](projections.md) | any doc | ALL projections off one model — md/llmtext/context/tiddlers/okf/distill/report/compare/inspect/scikgtex/stex/lean |

New here? Read [../QUICKSTART.md](../QUICKSTART.md) first (60-second workflow), then
[../SKILL.md](../SKILL.md) for the full command surface. Driving pdfdrill as an AI
agent? Read [../AGENTS.md](../AGENTS.md) — the DO/NEVER rules and allowed transitions.
